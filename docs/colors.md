## Gaming Community Considerations

### Dark Theme as Primary Experience
This color system is optimized for **dark theme as the default** to match gaming community preferences:

- **Deep Backgrounds**: True blacks and rich grays reduce eye strain during extended gaming sessions
- **Vibrant Accents**: Higher contrast accent colors (#B385B3, #E6B894) ensure UI elements pop against dark backgrounds  
- **Gaming Aesthetics**: Plum and burgundy tones create a sophisticated, non-generic gaming feel
- **Accessibility**: Enhanced contrast ratios (21:1, 12.6:1) support long gaming sessions

### Performance in Both Themes

## Accessibility Guidelines

### Contrast Requirements
All color combinations must meet WCAG 2.1 standards:
- **AA Normal Text**: 4.5:1 minimum contrast ratio
- **AA Large Text**: 3:1 minimum contrast ratio
- **AAA Normal Text**: 7:1 minimum contrast ratio
- **AAA Large Text**: 4.5:1 minimum contrast ratio

### Tested Combinations
The following combinations meet WCAG AA standards:

#### Text on Backgrounds
- `--color-text-primary` on `--color-background` (21:1 ratio)
- `--color-text-secondary` on `--color-background` (12.6:1 ratio)
- `--color-on-primary` on `--color-primary` (5.8:1 ratio)
- `--color-on-secondary` on `--color-secondary` (4.9:1 ratio)

#### Interactive States
- Focus indicators use `--color-border-focus` with 3px outline for visibility
- Hover states provide 10% luminance shift for clear feedback
- Active states provide 20% luminance shift for definitive response

## Usage Guidelines

### Component Application
- **Buttons**: Use `--color-primary` for primary actions, `--color-secondary` for secondary actions
- **Links**: Use `--color-primary` with `--color-primary-hover` for hover states  
- **Backgrounds**: Layer using `--color-background`, `--color-surface`, `--color-surface-raised`
- **Text Hierarchy**: Primary for headings, secondary for body, tertiary for captions
- **Borders**: Use `--color-border` for subtle divisions, `--color-border-hover` for interactive elements

### State Communication
- **Success**: Confirmations, completed actions, positive feedback
- **Warning**: Cautions, reversible actions, attention needed
- **Error**: Failures, validation errors, critical issues
- **Info**: Neutral information, tips, general notifications

2. Test all new combinations against WCAG standards
3. Ensure semantic roles remain clear and distinct
4. Document any new color roles or overrides
