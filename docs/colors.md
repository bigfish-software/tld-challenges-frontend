## Gaming Community Considerations

### Dark Theme as Primary Experience
This color system is optimized for **dark theme as the default** to match gaming community preferences:

- **Deep Backgrounds**: True blacks and rich grays reduce eye strain during extended gaming sessions
- **Vibrant Accents**: Higher contrast accent colors (#B385B3, #E6B894) ensure UI elements pop against dark backgrounds  
- **Gaming Aesthetics**: Plum and burgundy tones create a sophisticated, non-generic gaming feel
- **Accessibility**: Enhanced contrast ratios (21:1, 12.6:1) support long gaming sessions

### Performance in Both Themes
```css
/* Light theme optimizations */
.gaming-card-light {
  background: var(--color-surface-100);    /* Subtle warm background */
  border: 1px solid var(--color-neutral-200); /* Soft borders */
  color: var(--color-text-primary);        /* High contrast text */
}

/* Dark theme optimizations */  
.gaming-card-dark {
  background: var(--color-neutral-800);    /* Rich dark surface */
  border: 1px solid var(--color-neutral-700); /* Subtle borders */
  color: var(--color-text-primary);        /* Pure white text */
  box-shadow: 0 4px 6px rgba(0, 0, 0, 0.3); /* Enhanced depth */
}
```

### Theme Transition Smoothness
The 50-900 scale structure ensures smooth theme transitions:
- **Light to Dark**: Semantic roles maintain meaning while colors invert appropriately
- **Consistent Branding**: Primary plum (#4A2C4A → #B385B3) maintains brand recognition
- **Gaming Community Appeal**: Dark theme default aligns with community preferences

## Accessibility Guidelines

### Contrast Requirements
All color combinations must meet WCAG 2.1 standards:
- **AA Normal Text**: 4.5:1 minimum contrast ratio
- **AA Large Text**: 3:1 minimum contrast ratio
- **AAA Normal Text**: 7:1 minimum contrast ratio
- **AAA Large Text**: 4.5:1 minimum contrast ratio

### Tested Combinations
The following combinations meet WCAG AA standards:

#### Text on Backgrounds
- `--color-text-primary` on `--color-background` (21:1 ratio)
- `--color-text-secondary` on `--color-background` (12.6:1 ratio)
- `--color-on-primary` on `--color-primary` (5.8:1 ratio)
- `--color-on-secondary` on `--color-secondary` (4.9:1 ratio)

#### Interactive States
- Focus indicators use `--color-border-focus` with 3px outline for visibility
- Hover states provide 10% luminance shift for clear feedback
- Active states provide 20% luminance shift for definitive response

## Usage Guidelines

### Component Application
- **Buttons**: Use `--color-primary` for primary actions, `--color-secondary` for secondary actions
- **Links**: Use `--color-primary` with `--color-primary-hover` for hover states  
- **Backgrounds**: Layer using `--color-background`, `--color-surface`, `--color-surface-raised`
- **Text Hierarchy**: Primary for headings, secondary for body, tertiary for captions
- **Borders**: Use `--color-border` for subtle divisions, `--color-border-hover` for interactive elements

### Gaming-Specific UI Patterns
```css
/* Challenge difficulty indicators */
.difficulty-easy { 
  background: var(--color-success-600);
  color: var(--color-on-success);
}
.difficulty-medium { 
  background: var(--color-warning-500);
  color: var(--color-on-warning);
}
.difficulty-hard { 
  background: var(--color-error-600);
  color: var(--color-on-error);
}

/* Tournament status indicators */
.tournament-active {
  background: var(--color-primary);
  color: var(--color-on-primary);
  border: 2px solid var(--color-primary-hover);
}

/* Gaming card elevated surfaces */
.challenge-card {
  background: var(--color-surface);
  border: 1px solid var(--color-border);
  transition: all 0.2s ease;
}
.challenge-card:hover {
  background: var(--color-surface-raised);
  border-color: var(--color-border-hover);
  transform: translateY(-2px);
}
```

### State Communication
- **Success**: Confirmations, completed actions, positive feedback
- **Warning**: Cautions, reversible actions, attention needed
- **Error**: Failures, validation errors, critical issues
- **Info**: Neutral information, tips, general notifications

### Extension Rules
When extending this palette:
1. Maintain the 50-900 scale structure for consistency
2. Test all new combinations against WCAG standards
3. Ensure semantic roles remain clear and distinct
4. Document any new color roles or overrides
5. Validate across both light and dark themes

### Theming Overrides
Subdomains or specialized themes may override specific values:
```css
/* Example: Brand-specific primary override */
:root[data-theme="subdomain"] {
  --color-primary-500: #custom-brand-color;
  /* Recalculate dependent shades accordingly */
}
```

## Implementation Notes

### CSS Integration
Include this color system in your root CSS:
```css
:root {
  /* Insert color variables here */
}

[data-theme="dark"] {
  /* Insert dark theme overrides here */
}
```

### Component Library Integration
Reference semantic roles in component styles:
```css
.button {
  background-color: var(--color-primary);
  color: var(--color-on-primary);
  border: 1px solid var(--color-border);
}

.button:hover {
  background-color: var(--color-primary-hover);
}
```

This color system provides the foundation for consistent, accessible, and maintainable UI design across the component library and associated web properties.
